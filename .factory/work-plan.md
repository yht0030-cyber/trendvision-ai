## 五、人工标注 → 全自动 路线图 (2026-06-19)

### 标注范围（三大核心工具）
用户偶尔在图上手工画线，标注以下任意一项或多项：
1. **Qjt 区间**（白线）：上升/下降区间的起止时间
2. **趋势线**（斜线）：关键高/低点的连线
3. **拐点线**（通道平行线）：外延线 + 平行下移/上移

### 第一阶段：人工标注 + 文字传递（当前）
- 用户画线 -> 文字告知位置 -> 助手注入分析引擎对比脚本输出
- 下游信号（波浪/A点/评分）全自动
- 每次标注自动存档为训练数据

### 第二阶段：视觉模型直读（换模型后立即启用）
- 用户发 K 线截图 -> 视觉模型自动提取所有标注（区间/趋势线/拐点线）
- 零文字描述，一张图搞定
- 前提：切换到 Claude/GPT-4o/Gemini 等支持图像的模型

### 第三阶段：全自动（数据积累足够后）
- 收集 >= 20 组人工标注 vs 脚本计算结果
- 修复 calc_qjt() / find_trendline() / find_channel()
- 验证标准：Qjt 准确率 >= 85%，趋势线 >= 80%，拐点线 >= 90%
- 成功后用户只发截图（甚至不发），全自动跑

### 痛点：为什么还不能帮用户交易

**核心问题：只能分析，不能执行和监控。**

| 缺失能力 | 影响 |
|---------|------|
| 没有自动盯盘 | 不能实时监控价格变化，不会主动通知 |
| 没有自动开仓 | 没对接交易所下单API，分析完了要用户自己去币安操作 |
| 没有飞书推送 | 接近关键位时不会自动推送消息 |
| Qjt 有 bug | ETH A点准确率仅11%，下降区间漏检 |
| 没有一键分析 | 每次要手动打命令，不是自动推送 |
| 读不了截图 | 用户手画的 K 线图识别不了 |

### 完善建议（按优先级）

#### 🔴 P0：加监控脚本 monitor.py（最大变数）
在 trend_analyzer.py 基础上包装一个监控模式：
- 每 30 分钟自动拉一次数据
- 检测到评分 ≥ 5 的信号时 → 飞书推送通知
- 价格接近趋势线/拐点线/分界点A时 → 推送预警
- 核心：让系统主动找你，而不是等你找它

#### 🟡 P1：修 Qjt 两个 bug（地基问题）
- Qjt 趋势转向识别（下降区间完全漏检）
- Qjt 外/内区间嵌套去重（ETH A点 11%）

#### 🟢 P2：简化输出
- 每次分析只给结论："📊 评分 8/10，方向：做多，入场：XXX，止损：XXX"
- 不要输出一长串数据，用户要的是操作建议

#### 🔵 P3：实盘对接（半自动，慎重）
- 系统给信号+入场位+止损位 → 用户手机手输开单
- 不平仓自动开，只给建议
- 平仓后告诉系统盈亏 → 自动记录复盘

### 最值得先做 3 件事
1. **修 Qjt 趋势转向 bug** → ETH 恢复正常，整体信号质量提升
2. **加 monitor.py 监控脚本** → 自动飞书推送，不用手动问行情
3. **优化输出格式** → 打开飞书 3 秒看懂能不能开单
