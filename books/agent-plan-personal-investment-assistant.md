# Agent Plan CookBook -「个人投资助手」 Claude Code

> 本案例演示如何基于 Agent Plan + Claude Code + Data MCP + 联网搜索 Harness，搭建一个面向普通投资者的个人投资助手网站。
>
> 本案例的重点不是一次性生成一个静态网页，而是通过 Claude Code 完成一个可持续迭代的网站开发过程，并将开发过程沉淀为可复用的网站开发 Skill。后续用户可以继续调用这个 Skill，对页面结构、Data MCP 调用、引用来源展示、历史记录和风险摘要等能力进行调试和扩展。
>
> 网站生成后，用户可在其中沉淀自己关注的个股、每只个股的研究偏好、历史简报和盘后风险摘要。系统基于 Data MCP 股票金融数据和 Agent Plan 联网搜索 Harness 生成个股最新简报，并保留引用来源，进行风险监控，方便用户后续回看、核验和持续跟踪。

## 一、项目介绍

### 核心使用能力

| 能力点 | 说明 |
| --- | --- |
| 语言模型 | Agent Plan 集合优秀主流国产模型，可灵活自由切换或使用 Auto 模式调度。通过优秀编程能力高效完成优质网站交付 |
| Claude code / Agent 能力 | 承接自然语言需求理解、网站方案生成、任务编排、前后端开发、数据调用、页面交付和后续调试优化。 |
| 联网搜索 | Agent Plan 接入与豆包/头条同源的联网搜索能力（BETA 版），可实时检索互联网公开域内容（新闻、资讯、图文等），结合语言模型完成信息整合。支持通过 Skill 或 MCP 方式接入使用，用于补充公开新闻、行业动态、政策变化、舆情事件和风险信息。 |
| Data MCP：股票金融数据 / 国内企业工商数据 | 股票金融数据包括国内股票行情、财务指标、公告数据。覆盖 A 股全市场，支持按公司、时间、指标类型多维查询，返回结构化字段，数据来源可追溯。国内企业工商数据覆盖国内全量企业，可提供基本信息、经营状况、知识产权等维度的企业工商数据的数据集。 |
| 多工具兼容 | 可在 Claude Code、OpenClaw、ArkClaw、Hermes Agent 等主流 Agent 平台中使用，无需重复开发或额外配置。 |
| 消耗统一计量 | 所有操作（问答、数据查询、Skill 生成、定时任务）统一计费，可在控制台实时查看消耗和权限使用情况。 |

## 二、前置准备

### Step1 购买 Agent Plan 套餐

请登录「火山方舟」或「官网活动页」，购买 Agent Plan 套餐。

建议购买 Medium 及以上套餐，以获得更完整的 Agent 模型与 Harness 使用体验。Small 套餐更适合基础测试和体验。

### Step2 快速配置

需要获取 Agent Plan 支持的专属 Base URL、专属 API Key 及模型配置，才能正常使用并抵扣套餐额度。

推荐使用 [Ark Helper](https://www.volcengine.com/docs/82379/2373742?lang=zh#60f11edc) 一键安装模型及 Data MCP。以下为该项目所使用的配置信息，您可按需选择。

| 方式 | 配置项 | 说明 |
| --- | --- | --- |
| 方式1: 自动化配置（推荐） | Ark Helper | Ark Helper 是一个编码工具助手，支持快速配置选择的工具接入 Agent Plan。将自动完成语言模型、向量模型 Skill、联网搜索 Skill / MCP Data MCP 的安装，无需手动安装。具体参照[官网文档](https://www.volcengine.com/docs/82379/2373742?lang=zh#60f11edc) |
| 方式2: 手动配置 | 配置模型及专属 Base URL | 获取 Agent Plan 的专属 Base URL、模型配置。套餐集合优秀主流国产模型，可灵活自由切换或使用 Auto 模式调度。详见[快速开始](https://www.volcengine.com/docs/82379/2373738?lang=zh) |
| | 获取专属 API Key | 请注意 Agent Plan 使用专属 API Key，请勿混用。 |
| | 安装 Data MCP | ```claude mcp add --transport http datapro https://datapro.hqd.cn-beijing.volces.com/mcp --header "X-Agent-Plan-Key: <Agent-Plan-Key>"``` 安装 Data MCP，国内股票金融数据、国内企业工商数据、国内论文搜索三类专业数据整合为统一的"专业数据搜索服务" |
| | 安装联网搜索 Skill / MCP | 可在控制台「配置 Harness」处领取联网权益，订阅套餐即可每月赠送一定免费额度，请注意需要从控制台获取联网搜索 API Key。具体参照[联网搜索配置文档](https://www.volcengine.com/docs/82379/2301412?lang=zh) |

### Step3 在 Claude Code 中使用

完成配置后，可在主流编程及 Agent 工具中接入 Agent Plan，使用模型、Data MCP 和联网搜索 Harness 完成网站开发。截图演示以 Claude Code 为例，接入方舟 Agent Plan 服务进行使用。在正式开发网页前，需要先确认 Data MCP 和联网搜索 Harness 已经接入 Claude Code 并可以正常调用。

## 三、详细操作指南

### 1. 网站开发流程

#### Step1 自然语言表达投研需求

用户在 Claude Code 上说明自己的关注股票和使用需求。例如：

> 我想做一个个人投资助手网站，关注比亚迪、宁德时代、科大讯飞这几只股票，主要想快速了解它们最近有没有值得关注的变化，同时不想每天手动查公告和新闻。

Claude Code 应识别出两类核心需求：

| 需求类型 | 用户表达 | 对应能力 |
| --- | --- | --- |
| 主动查看 | 想快速了解某只股票最近有没有值得关注的变化 | 个股最新简评 |
| 持续跟踪 | 不想每天手动查公告、新闻和风险事件 | 盘后风险监控 |

#### Step2 沉淀关注股票和研究偏好

用户可以继续用自然语言补充自己的研究偏好。比如：

> 比亚迪我关注销量、利润变化和新能源车竞争情况；宁德时代关注财务表现、行业变化、海外风险和原材料价格；科大讯飞关注 AI 业务、政策风险、大模型进展和近期公告。

Claude Code 应将用户偏好整理为结构化内容：

| 股票 | 关注方向 |
| --- | --- |
| 比亚迪 | 销量、利润变化、新能源车竞争 |
| 宁德时代 | 财务表现、行业变化、海外风险、原材料价格 |
| 科大讯飞 | AI 业务、政策风险、大模型进展、近期公告 |

这一步会沉淀为用户个人投研偏好。后续网站生成后，这些偏好应直接体现在首页和个股详情页中。

#### Step3 生成网站方案

在理解用户需求和关注偏好后，Claude Code 应先输出网站方案。

本案例要求生成的不是纯静态页面，而是一个可运行的真实前后端 Demo：

- 前端展示首页、个股简评详情页、盘后风险摘要详情页；
- 后端负责调用 Data MCP 和联网搜索 Harness；
- 简评和风险摘要均保存历史记录；
- 结果保留引用来源。

建议方案包括：

| 模块 | 说明 |
| --- | --- |
| 首页 | 展示今日要点、用户关注的股票清单、每只股票的研究偏好，以及两个核心入口：「个股简评」「盘后风险摘要」。 |
| 个股简评详情页 | 用于用户主动查看某只股票近期是否值得关注。展示 Data MCP 专业数据事实、公开信息补充、引用来源和历史简评。 |
| 盘后风险摘要详情页 | 用于持续监控某只股票的公告、新闻和风险事件。展示盘后监控配置、本次风险摘要、引用来源、历史风险摘要。 |
| 开发 Skill / 调试记录 | 不作为功能首页模块展示，仅用于记录开发过程、接口设计、Data MCP 调用方式、联网搜索调用方式、历史记录规则。 |

#### Step4 开发网站并沉淀开发 Skill

Claude Code 完成方案确认后，再开始生成网站。

**最低要求**包括：

- 前端点击「个股简评」时，真实请求后端接口；
- 前端点击「盘后风险摘要」或「立即执行一次」时，真实请求后端接口；
- 后端优先调用 Agent Plan 的 Data MCP 查询专业数据；
- 当 Data MCP 信息不足时，后端再调用 Agent Plan 的联网搜索 Harness 补充公开新闻和公告；
- 后端返回结果、引用来源和来源类型，前端只展示用户可理解的内容；
- 页面不展示接口日志、query 日志、调用次数、API Key 和 AFP 等技术信息。

**建议数据调用时输入规范提示词：**

```
请在调用数据时保证数据准确。

时间范围：
- 以当前日期 {{当前日期}} 为准；
- 只使用 {{当前日期}} 及之前已经公开披露的数据；
- 财务数据默认取截至当前日期的最新正式披露期；
- 新闻、公告、销量等默认取最近 30 天数据，除非另有指定。

数据规则：
1. 先查 Data MCP，查不到或信息不足时再用联网搜索补充。
2. Data MCP 和联网搜索结果要分开展示，不能混用。
3. 每个数字展示前，必须确认时间、单位、指标口径。
4. 同比/环比必须确认比较对象，不能把子指标增速写成整体增速。
5. 联网搜索内容必须和当前公司直接相关，并保留来源和可跳转链接。
6. 查不到或无法确认的数据不要展示，不要用 mock、历史记录或旧缓存替代。
```

Claude Code 完成网站开发后，用户可以要求其启动本地服务并返回访问地址。

随后，Claude Code 需要将本次开发过程沉淀成一个可复用 Skill。该 Skill 至少包括：

| Skill 内容 | 说明 |
| --- | --- |
| 页面结构 | 记录首页、个股简评详情页、盘后风险摘要详情页的结构。 |
| 开发 Prompt | 记录本次用于生成网站的核心自然语言输入和开发约束。 |
| 前后端接口 | 记录 stocks、brief、monitor、history、usage-log 等接口设计。 |
| Data MCP 调用方式 | 记录 Data MCP 的查询策略、结构化结果解析方式、数据不足时的处理规则。 |
| 联网搜索 Harness 调用方式 | 记录联网搜索的触发条件、查询方式、返回结果处理方式。 |
| 数据优先级 | 固定为 Data MCP 优先，联网搜索 Harness 补充公开信息。 |
| 引用来源规则 | 记录正文脚标、来源列表、来源类型和链接展示规则。 |
| 历史记录规则 | 记录个股简评和盘后风险摘要的保存方式。 |
| 调试记录 | 记录后端解析错误、Data MCP 返回异常、联网搜索配置错误、引用来源缺失等问题及修复方式。 |
| 验收标准 | 记录页面、数据、交互、引用和 AFP 用量的验收口径。 |

后续如果用户说"帮我继续优化个人投资助手网站"，Claude Code 应调用这个网站开发 Skill，基于已有结构继续修改，而不是重新生成一个全新项目。

### 2. 网站使用流程

#### Step1 查看首页

用户进入网站首页后，可以看到：

- 我的关注股票；
- 每只股票的研究偏好；
- 「个股简评」入口；
- 「盘后风险摘要」入口。

#### Step2 查看个股简评

用户点击「个股简评」后，系统优先调用 Data MCP 查询该股票的专业数据事实，例如销量、财务指标、公告、经营数据等，如果 Data MCP 返回信息不足，系统再调用联网搜索 Harness 补充公开新闻和行业信息。

个股简评详情页应展示：

- Data MCP 专业数据事实；
- 公开信息补充；
- 引用来源；
- 历史简评。

> 个股简评不能只使用前端的示例文案。用户点击「个股简评」后，应由前端请求后端接口，后端实际调用 Data MCP 获取专业数据；如 Data MCP 返回不足，再调用联网搜索补充公开信息。只有完成真实接口调用后，才能用于验证 Data MCP / 联网搜索的能力效果和 AFP 消耗。

#### Step3 查看盘后风险摘要

用户可以为关注股票配置盘后风险摘要的定时监控任务。

配置项包括：

- 监控对象；
- 监控频率；
- 监控时间；
- 检查内容；
- 下次执行时间。

> 盘后风险摘要同样需要接入真实后端。用户点击「盘后风险摘要」或「立即执行一次」时，应触发后端任务，由后端查询公告、新闻、风险事件等信息，并返回本次风险摘要、引用来源和历史记录。若仅展示固定风险文案，则不能作为真实 Harness 调用结果。

#### Step4 调用网站开发 Skill 持续优化

如果用户后续希望调整网站，例如：

- 修改页面布局；
- 增加引用来源详情；
- 修复 Data MCP 数据返回异常；
- 修复联网搜索 Harness 配置；
- 优化盘后风险摘要；
- 增加历史记录筛选；
- 优化 AFP 用量统计；

可以直接对 Claude Code 说：

> 请调用「个人投资助手网站开发 Skill」，基于当前网站继续优化 skill。

Claude Code 应根据已沉淀的网站结构、接口设计、调试记录和验收标准继续修改。

### 3. 查询 AFP 用量消耗

用户可通过方舟控制台查询 Agent Plan 套餐 AFP 的实时用量。本案例主要消耗来自语言模型调用、Data MCP 股票金融数据查询、联网搜索调用、Skill 生成和定时任务执行。实际消耗会受到模型选择、对话轮次、数据查询次数和定时任务频率影响，最终以控制台实际展示为准。详情可见 [AFP 积分机制](https://www.volcengine.com/docs/82379/2366394?lang=zh#c90d28c2)。

需要注意的是，后端联调阶段包含真实后端接入、Data MCP 鉴权修复、联网搜索 Harness 调用、接口调试、curl 验证和前端点击验证，根据用户的实际情况会产生不同的 AFP 消耗。

| 消耗项 | 消耗说明 | 预估 AFP 消耗 |
| --- | --- | --- |
| 语言模型 | 完成需求理解、页面方案、前端页面生成、本地运行、后端接入、调试和 Skill 沉淀 | 2000 AFP |
| Data MCP | 查询历史行情、财务、公告、经营相关指标 | 65 AFP（调用 8 次） |
| 联网搜索 Harness | Data MCP 信息不足时补充新闻、公告、行业变化和风险事件 | 免费（现阶段） |

> 说明：本次结果仅代表当前账号和本次测试路径下的用量情况，只包括网站开发，后续网站使用未计入消耗。不同用户的需求描述、页面复杂度、调试次数、模型选择、Data MCP 查询次数、联网搜索调用次数不同，实际 AFP 消耗会有差异，最终以方舟控制台实际用量为准。

## Claude Code vs OpenClaw 开发体验对照

| 对比项 | Claude Code | OpenClaw |
| --- | --- | --- |
| 复杂开发与连续迭代 | 适合连续开发：能理解需求、改代码、接后端、启动服务、调试问题 | 不太适合稍微复杂的开发任务，长流程容易断 |
| 上下文与配置稳定性 | 项目文件、需求、模型配置、Data MCP / 联网搜索配置更容易保持连续和稳定 | 长流程中容易忘记前面需求；一旦卡死、回滚或环境重置，已配置的模型和 MCP 可能失效 |
| 开发效率 | 同样流程通常可以较快推进，适合快速生成、验证和迭代 | 复杂任务中频繁重启、回滚，原本几分钟的流程可能被拉长到数小时 |
| 代码修改与后端联调 | 能直接改前端、后端、package、接口逻辑，并运行验证 | 复杂修改时更容易卡住或回滚，后端链路一复杂稳定性下降明显 |
| 开发过程透明性 | 显示进度和代码调整 | 过程不透明 |
| 本地部署与稳定访问 | 文件在用户本地项目目录中，可直接启动本地服务，例如 `npm run dev`，通过本地链接稳定访问 | 生成文件通常在 OpenClaw 自己的文件夹里而非本地，后端跑在云端的沙箱容器里；页面访问往往依赖隧道链接，隧道容易断或失效 |
| 适合场景 | 网站开发、代码生成、后端联调、持续迭代 | 轻量体验、简单问答、简单 Agent 使用 |
